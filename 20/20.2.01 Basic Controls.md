# Control Systems in Robotics

> [!NOTE]
> ### INFO DUMP: Control Systems 
> 
> #### 1. Open-Loop Control
> 
> 
> In an **Open-Loop** system, the controller sends a command to the robot without receiving any feedback. The robot executes an instruction without checking if the goal was actually achieved.
> * **Real-World Example:** A standard microwave. If you set it for 2 minutes, it runs for 2 minutes. It does not know if the food is cooked; it simply follows the timer.
> * **Use Case:** Simple, predictable tasks where the environment doesn't change.
> * **Weakness:** The robot cannot correct for errors (like a wheel slipping) or adapt to obstacles.
> 
> 
> #### 2. Closed-Loop Control
> 
> 
> **Closed-Loop Control** (feedback control) uses sensors to adjust behavior dynamically. The robot constantly compares its **current state** against its **desired state** and adjusts to bridge the gap.

> * **Real-World Example:** A self-driving car. It uses cameras to see lane lines. If it drifts, the sensors detect the error, and the controller adjusts the steering to center the vehicle.
> * **Our Objective:** We will implement **discrete control** (On/Off logic) to process sensor data for precise robot actions.
> 
>

## Level 3: On-Off Closed-Loop Control

On-off control is a binary feedback system. Think of a **Thermostat**:

* If Temp < Target: Heater **ON**.
* If Temp > Target: Heater **OFF**.

### The Deadband 

If we tell a robot to stop at *exactly* 10cm, it will jitter (oscillate) back and forth. This happens because sensor readings fluctuate slightly (e.g., 9.9cm to 10.1cm). To fix this, we create a **Deadband**, a small "safe zone" where the robot stays still.

> [!TIP]
> **Logic Formula:**
> * **Too far?** `distance > (target + tolerance)` $\rightarrow$ Move Forward.
> * **Too close?** `distance < (target - tolerance)` $\rightarrow$ Move Backward.
> * **In the zone?** $\rightarrow$ Stop.
>
