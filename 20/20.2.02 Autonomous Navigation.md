# Autonomous Navigation

In this lab, you will move from using a single ultrasonic sensor to a dual IR (Infrared) sensor setup. This allows your robot to "see" in two directions at once, enabling smarter decision-making.

> [!NOTE]
> **IR vs. Ultrasonic:** Unlike the Ultrasonic sensor which measures distance in cm, these IR sensors are digital. They return a **1** (Obstacle Detected) or a **0** (Path Clear). You set the detection distance physically using the potentiometer on the sensor.

---

## Level 1: The Virtual Bumper (On-Off Control)

Your first goal is to recreate "Stop" behavior using two sensors instead of one.

* **Setup:** Mount two IR sensors to the front of your robot (Left and Right) using the brass hex offsets.
* **The Logic:** Program the robot to drive forward toward a wall and stop if an obstacle is detected.

> [!TIP]
> **Coding Tip:** Create variables for `Left_Sensor` and `Right_Sensor`. Use an **OR** block (represented as `||` in Ardunio) so that if **either** sensor detects the wall, the robot stops.

```cpp
// Logic Example:
if (Left_Sensor == 1 || Right_Sensor == 1) {
    set_speed(0, 0); // Stop
} else {
    set_speed(1, 1); // Drive Forward
}

```

## Level 2: Making a Choice (Left vs. Right)

Now, let’s give the robot the ability to steer away from trouble rather than just stopping.

* **The Goal:** Program the robot to dodge obstacles.
* If the **Left** sensor is blocked → Turn **Right**.
* If the **Right** sensor is blocked → Turn **Left**.


* **Hardware Calibration:** Use a small screwdriver to adjust the potentiometer (the gold/blue dial) on the back of each IR sensor. This sets the detection distance.
* **Refinement:** Ensure your "Turn" command is sharp enough to actually clear the obstacle!

## Level 4: Two Paths Diverge in a Forest

It’s time to put all your logic together for the final challenge. You must develop a fully autonomous obstacle navigation behavior.

* **The Challenge:** Navigate a "forest" of blocks placed randomly on the floor.
* **Requirements:** Your robot must drive autonomously and handle three scenarios:
1. **Drive Forward:** Do this by default when the path is clear.
2. **Avoidance:** Dodge individual blocks (Level 2 logic).
3. **Escape:** Resolve "dead-ends" or corners using random turns (Level 3 logic).
