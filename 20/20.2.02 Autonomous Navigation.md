# Autonomous Navigation

In this lab, you will move from using a single ultrasonic sensor to a dual IR (Infrared) sensor setup. This allows your robot to "see" in two directions at once, enabling smarter decision-making.

> [!NOTE]
> **IR vs. Ultrasonic:** Unlike the Ultrasonic sensor which measures distance in cm, these IR sensors are digital. They return a **1** (Obstacle Detected) or a **0** (Path Clear). You set the detection distance physically using the potentiometer on the sensor.

---

## Getting Started: Code Foundations

Before writing the logic for, you must tell the Arduino where the sensors are plugged in and how to talk to them.

### 1. Define your Pins

At the very top of your sketch (above `void setup()`), create constant variables for your pins. This makes the code easier to read and change later.

```cpp
const int LEFT_IR_PIN = 2;   // IR sensor on Digital Pin 2
const int RIGHT_IR_PIN = 3;  // IR sensor on Digital Pin 3

```

### 2. Configure the Pins

Inside `void setup()`, tell the Arduino that these pins are **INPUTS** (because data is coming *from* the sensor *to* the board).

```cpp

// This is were your your motors are Initialized
  pinMode(LEFT_IR_PIN, INPUT);
  pinMode(RIGHT_IR_PIN, INPUT);
```

### 3. Reading the Data

Inside `void loop()`, you need to constantly "read" the state of the pins and store those values in variables.
* remember this is just an example, add this to your previous assignments code. 

```cpp
  int leftState = digitalRead(LEFT_IR_PIN);
  int rightState = digitalRead(RIGHT_IR_PIN);
  
  // Now you can use 'leftState' and 'rightState' in your logic below


```

## Level 1: The Virtual Bumper aka On-Off Control

Your first goal is to recreate "Stop" behavior using two sensors instead of one.

* **Setup:** Mount two IR sensors to the front of your robot (Left and Right) using the brass hex offsets.
* **The Logic:** Program the robot to drive forward toward a wall and stop if an obstacle is detected.

> [!TIP]
> **Coding Tip:** Use an **OR** operator (represented as `||`) so that if **either** sensor detects the wall, the robot stops.

```cpp
// Level 1 Logic:
if (leftState == 1 || rightState == 1) {
    set_speed(0, 0); // Stop
} else {
    set_speed(1, 1); // Drive Forward
}

```

## Level 2: Making a Choice (Left vs. Right)

Now, let’s give the robot the ability to steer away from trouble rather than just stopping.

* **The Goal:** Program the robot to dodge obstacles.
* If the **Left** sensor is blocked: Turn **Right**.
* If the **Right** sensor is blocked: Turn **Left**.
* **Hardware Calibration:** Use a small screwdriver to adjust the potentiometer (the gold/blue dial) on the back of each IR sensor. This sets the detection distance.
* **Refinement:** Ensure your "Turn" command is enough to actually clear the obstacle

## Level 3: A Random Walk Down Wall Street

**The Problem:** What happens if the robot drives straight into a corner and **both** sensors see a wall at the same time?

* **TODO:** If both IR sensors detect an obstacle, both sides are blocked, the robot must choose a direction randomly to break the deadlock.

> **Logic for Deadlock:**
> 1. If (leftState == 1 **&&** rightState == 1)
> 2. Pick a random number: `int choice = random(0, 10);`

## Level 4: Two Paths Diverge in a Forest

It’s time to put all your logic together for the final challenge. You must develop a fully autonomous obstacle navigation behavior.

* **The Challenge:** Navigate a "forest" of blocks placed randomly on the floor.
* **Requirements:** Your robot must drive autonomously and handle three scenarios:
1. **Drive Forward:** Do this by default when the path is clear.
2. **Avoidance:** Dodge individual blocks (Level 2 logic).
3. **Escape:** Resolve "dead-ends" or corners using random turns (Level 3 logic).
