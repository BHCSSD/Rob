# Introduction to Odometry: Measuring Distance

**Odometry** is the method robots use to estimate their position by tracking wheel rotations. It allows your robot to count wheel rotations and translate them into physical distance (centimeters).

> [!TIP]
> **Real Life:** Imagine walking in a pitch-black room where you cannot see the walls. You can estimate how far you have traveled by counting your steps. In robotics, those "steps" are recorded by **Wheel Encoders** as **ticks** or **pulses**.

---

## 1. Understanding Your Hardware

Hobby motors usually utilize a slotted encoder disk and an Infrared (IR) speed sensor. The disk breaks the IR beam as it spins, sending a digital pulse to the Arduino.

### Key Terminology

* **Ticks (n):** A single pulse detected by the Arduino.
* **Ticks Per Revolution (TPR):** The number of slots in your encoder disk (usually 20 for basic hobby kits).
* **Gear Ratio:** If your encoder is on the motor shaft before the gearbox, you must account for the reduction (e.g., 48:1).
  * ours is not, so we are good   

---

## 2. The Math: Ticks to Centimeters

Use these three formulas to convert digital pulses into real-world distance:

### Step A: Wheel Circumference (C)

The distance the robot moves in one full rotation.
**C = 3.14 * Diameter**

### Step B: Distance Per Pulse (Dpp)

The "resolution" of your robot. This is how far the robot moves for every single tick detected.
**Distance Per Pulse (Dpp) = C / TPR**

### Step C: Calculating Travel

Multiply your current tick count (n) by your resolution (Dpp).
**Total Distance = n * Dpp**

---

## 3. Math Example
* **Wheel Diameter:** 6.5 cm
* **Encoder Disk:** 20 slots (mounted on the wheel shaft)
* **Current Ticks (n):** 30 pulses

**1. Circumference:** C = 3.14 * 6.5 
  * C=  20.41 cm

**2. Distance Per Pulse:** Dpp = 20.41 / 20 
  * Dpp = 1.02 cm/pulse
    
**3. Total Distance:** 30 ticks * 1.02 
  * **30.6 cm traveled**
---
## 4. Real-World Limitations

> [!WARNING]
> **The "Jitter" Factor:** > * **Slippage:** Yellow hobby wheels have limited traction. They often slip on smooth surfaces, meaning the encoder counts "movement" that did not actually occur.
> * **Backlash:** Hobby gearboxes have "slop" between the plastic gears. The motor might spin several degrees before the wheel starts turning.
> * **Mismatched Motors:** One hobby motor is almost always faster than the other. Use odometry to monitor both wheels and slow down the faster motor so the robot drives straight.
>

>  ### Troubleshooting the IR Sensor
> 
> If your `Serial.print` isn't showing any pulses:
> 
> * **Check the Gap:** Ensure the encoder disk is spinning freely through the center of the IR sensor "U" shape without hitting the sides.
> * **Indicator Lights:** Most IR sensors have a small LED that flickers when a slot passes through. If that light isn't flickering, your sensor isn't "seeing" the disk.
> * **Wiring:** Double-check that your `odoPin` in the code matches the physical pin on the Arduino (e.g., Pin 2).

## Starter Code

### 1. The `set_speed(left, right)` Function

This is your **Direction Controller**. Instead of writing four lines of `digitalWrite` every time you want to move, you use this shortcut.

* **How to use it:** Pass two numbers into the parentheses: `set_speed(Left Motor, Right Motor)`.
* **The Inputs:** * `1` = Forward
* `-1` = Backward
* `0` = Stop

  * **Example:** `set_speed(1, -1);` will make the left wheel go forward and the right wheel go backward, causing the robot to spin in place.

---

### 2. The `drive_distance(targetCm)` Function

This is your **Distance Controller**. It uses the IR sensor to "count" how far the robot has traveled and stops automatically when the goal is reached.

* **How to use it:** Put the distance you want (in centimeters) inside the parentheses.
* **The Logic:** 1. It calculates how many **Ticks** are needed ().
2. It turns the motors on using `set_speed(1, 1)`.
3. It enters a `while` loop that constantly checks the `odoPin`.
4. Once the count is reached, it calls `set_speed(0, 0)` to stop.
  * **Example:** `drive_distance(40);` will make the robot drive forward until the sensor counts 39 ticks.

---

### 3. Putting it together in `setup()`

Because these assignments are specific tasks (Level 1 and Level 2), you place the commands in `setup()` so they run once as soon as the robot is turned on.
* this is the only time we will use setup to code in. 

**Example for Level 2:**

```cpp
drive_distance(40); // Drive 40cm
set_speed(1, -1);   // Start turning
delay(600);         // Wait for turn to complete
set_speed(0, 0);    // Stop
drive_distance(20); // Drive remaining 20cm

```

### Important: The Blocking Nature

While `drive_distance` is running, the robot is stuck inside that `while` loop. It cannot do anything else (like check for obstacles) until it finishes counting the ticks. This is why we use it for specific, controlled distance challenges!

```cpp
// Motor control pins
const int leftMotorIn1 = 4;
const int leftMotorIn2 = 5;
const int rightMotorIn1 = 7;
const int rightMotorIn2 = 8;

// Odometry Sensor Pin
const int odoPin = 2; // you might need to change this

// Odometry Math (based on 6.5cm wheel and 20-slot disk) this might need to change
float dpp = 1.02; // Distance Per Pulse in cm

void setup() {
  Serial.begin(9600);
  
  pinMode(leftMotorIn1, OUTPUT);
  pinMode(leftMotorIn2, OUTPUT);
  pinMode(rightMotorIn1, OUTPUT);
  pinMode(rightMotorIn2, OUTPUT);
  
  pinMode(odoPin, INPUT);

  delay(2000); // Safety pause before moving

  // --- Level 1: Drive 40 cm ---

  
  // --- Level 2: L-Turn ---

}

void loop() {
  // All coding for the assignment is done in setup or helper functions
}

// Function to drive a specific distance using the IR sensor
void drive_distance(float targetCm) {
  int pulseCount = 0;
  bool lastState = LOW;
  int targetPulses = targetCm / dpp;

  // Start motors forward
  set_speed(1, 1);

  while (pulseCount < targetPulses) {
    bool currentState = digitalRead(odoPin);

    // State change detection (LOW to HIGH)
    if (currentState == HIGH && lastState == LOW) {
      pulseCount++;
      Serial.print("Pulses: ");
      Serial.println(pulseCount);
    }
    lastState = currentState;
  }

  // Stop once target is reached
  set_speed(0, 0);
}

// Set motor direction: 1 = forward, 0 = stop, -1 = backward
void set_speed(int leftDir, int rightDir) {
  // Left motor logic 
  if (leftDir == 1) {
    digitalWrite(leftMotorIn1, HIGH);
    digitalWrite(leftMotorIn2, LOW);
  } else if (leftDir == -1) {
    digitalWrite(leftMotorIn1, LOW);
    digitalWrite(leftMotorIn2, HIGH);
  } else {
    digitalWrite(leftMotorIn1, LOW);
    digitalWrite(leftMotorIn2, LOW);
  }

  // Right motor logic
  if (rightDir == 1) {
    digitalWrite(rightMotorIn1, HIGH);
    digitalWrite(rightMotorIn2, LOW);
  } else if (rightDir == -1) {
    digitalWrite(rightMotorIn1, LOW);
    digitalWrite(rightMotorIn2, HIGH);
  } else {
    digitalWrite(rightMotorIn1, LOW);
    digitalWrite(rightMotorIn2, LOW);
  }
}
```
